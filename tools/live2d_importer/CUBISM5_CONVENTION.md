# Cubism 5 Model Directory Convention

## Recommended structure
```
app/src/main/assets/live2d/cubism5_models/<model_id>/
├── model.model3.json       ← entry point (or <model_id>.model3.json)
├── <model_id>.moc3
├── <model_id>.physics3.json
├── <model_id>.cdi3.json
├── <model_id>.pose3.json   (optional)
├── textures/
│   └── texture_00.png
├── motions/
│   └── *.motion3.json      (optional)
├── expressions/
│   └── *.exp3.json         (optional)
├── profile.json            ← generated by VTS profiler
└── import_report.json      ← generated by checker
```

## Naming rules
- All filenames: ASCII only (a-z, 0-9, _, -, .)
- No Chinese characters, spaces, or special symbols
- model3.json internally references relative paths from the model directory

## Profile lookup order (viewer5)
1. `<modelId>_profile.json`  (e.g. `xingyue_shuimu_profile.json`)
2. `<modelId_without_safe>_profile.json`  (strips `_safe` suffix)
3. `profile.json`  (generic, recommended for new models)
4. Fallback to VTS-measured hardcoded parameters

## New model workflow
1. Place model files in `live2d/cubism5_models/<id>/`
2. Check integrity:
   ```
   python tools/live2d_importer/check_cubism5_model.py --model-dir <path>
   ```
3. Fix any issues reported in `import_report.json`
4. Import into VTube Studio, run VTS profiler:
   ```
   python tools/vts_profiler/vts_model_profiler.py --model-id <id> --model-dir <path> --out <path>/profile.json
   ```
5. Rename `.model3.json` to `model.model3.json` (optional, for convention)
6. `assembleDebug` — viewer5 auto-loads `profile.json`
